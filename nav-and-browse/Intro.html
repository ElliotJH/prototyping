GOV.UK Navigation and Browse page wireframes

Explore page options

 

We've chosen not to implement an 'explore' style page for now.

 

Rationale:

• Home page ought to be supporting this function anyway

• Its position between the home page and every page on the site makes for a weird IA

• We might, however, create a site map page

 

Options explored were as follows:

 

1. Squares - Too flat, no hierarchy

2. Hero section -  Not bad, try applying to section pages

3. Global nav panel

4. Site map style - hard to scan

5. Site map style - grid - We'll use if we decide to have a site map

 

Section page options

 

We've opted for a simple grid of subsection panels that link to section pages. We'll break up the monotony with a large section header and introduction. We're not going to do highlights or top pages or anything like that until we're sure that it will benefit users.

 

1. Hero section - We don't have the data to discriminate like this. Risk that people will think that's all the section contains

2. 'Most popular' panel - See above

3. 'In this section' panel

4. 'In this section' horizontal nav

5. Three level hierarchy - Again, too much discrimination of sections

6. Squares - Too flat

7. Simple list - Too simple, doesn't feel appropriate for this level

8. Simple list - horizontal - As above

9. Categorised list of all pages - Items at bottom of page will be neglected

10. Categorised list - JS enabled - Too fiddly. Risk people won't know how it works

 

 

Subsection page options

 

We've chosen to use a simple list format to reinforce the 'one step from the answer' feel. Related pages will be surfaced at the bottom of the list and labeled with the section they belong to.

 

1. Related pages on right - Inappropriate as the whole page is about related content

2. Related pages below - Better, and it makes sense as these pages are likely to be less relevant

3. Small parent section above title

4. Large parent section above title

5. No crumb

 

 

Breadcrumbs options
